    Solución de problemas de audiencia

    En este tema aprenderá a exportar datos de formularios de prospectos recopilados a través de experiencias de Galería.

    Habilitación del registro detallado

    Habilitar el registro detallado permite ver información relacionada con Audiencia en la consola del navegador. Esto es útil cuando se intenta depurar problemas de audiencia. El registro detallado se puede habilitar agregando bcverbose=1 a la URL de la página con el reproductor habilitado para Audience o configurando el plugin Audience con la "verbose": true opción:

    Los mensajes de registro se pueden ver en la consola.

    Problemas del formulario principal

    A continuación se presentan algunos problemas y preguntas comunes que hemos visto con los formularios de prospecto.

    ¿Por qué los formularios de plomo siguen mostrándose para las integraciones de Marketo Munchkin incluso después de que el formulario ha sido enviado?

    La integración de Munchkin no tiene una API que podamos usar para preguntar si el usuario es conocido o no. Dado que no sabemos si es un usuario conocido, siempre mostramos el formulario de prospecto. La integración de REST de Marketo ofrece API para que podamos determinar usuarios conocidos frente a usuarios desconocidos. Se recomienda cambiar a una integración de REST de Marketo y puede evitar este problema.

    Mi formulario de prospecto personalizado está configurado para redirigir al enviar. ¿Por qué veo la página a la que se redirige por un segundo o dos después de enviar el formulario?

    Tenemos que esperar hasta que la página a la que se redirige se haya cargado completamente antes de descartar el formulario de prospecto. Una solución es redirigir a una página que es ligera (no tiene muchas imágenes o JavaScript), o a la página en blanco que configuramos (ubicada en https://map.brightcove.com/lead/complete) específicamente para abordar este problema.

    Configuré mi formulario de prospecto con HTML de confirmación. ¿Por qué veo una pantalla negra durante un par de segundos después de la presentación?

    Asegúrese de cambiar el color del texto del HTML. El HTML de confirmación probablemente sea visible, simplemente no puede verlo porque el color del texto predeterminado es negro.

    ¿Por qué mi formulario de prospecto de HubSpot no desaparece después de enviarlo?

    Asegúrese de que el formulario de prospecto en HubSpot está configurado para redirigirse a http://map.brightcove.com/lead/complete. Si desea mostrar un mensaje de confirmación, configure esto dentro de Audience y no HubSpot.

    ¿Por qué no se muestra mi formulario de prospecto?

    Si el formulario de prospecto no se muestra, primero asegúrese de que se cumplan todos los criterios para mostrar un formulario de prospecto. Tenga en cuenta que los criterios que dependen de que se vean varios vídeos no funcionarán en las actualizaciones de página. Están pensados para casos de uso en los que el siguiente vídeo se pone automáticamente en cola en el mismo reproductor.

    1. Confirme que el plugin Audience se ha añadido al reproductor. Para obtener más información, consulte Creación de un reproductor habilitado para audiencia-habilitado.
    2. Confirme que la integración se ha configurado correctamente y se envía al complemento.
    3. Confirma que el formulario de prospecto se ha añadido a ese jugador. La respuesta a la llamada que realiza el plugin para su configuración debe contener información de visualización del formulario de prospecto bajo la propiedad LeadForm si se ha agregado un formulario de prospecto al reproductor.
    4. Confirme que el espectador aún no es un cliente potencial. El formulario de prospecto solo se mostrará si Audience ha identificado que el espectador no es ya un cliente potencial en la plataforma de automatización de marketing (MAP) o aún no ha rellenado un formulario de prospecto. El plugin realiza una llamada JSONP separada al servidor Audience para comprobar si el espectador es un cliente potencial. Si el visor es un cliente potencial, el servidor responde con un estado de 1 y cualquier otro identificador que pueda ser útil para sincronizar los datos de vista con el MAP. Si el visor no es un cliente potencial, el servidor responde con un estado de 0.

 JSONPCallback_2935650618 ({"status» :0});
      • Ejemplo de respuesta para un visor que es un cliente potencial: 
        /**/ tipode jsonpCallback_45936049288 === 'función' &&
 JSONPCallback_45936049288 ({"status» :1, "eloqua»: {"contactId» :"5", "InternalID» :"CTCPB000000000005"}});

    problemas de sincronización

    A continuación se muestran algunos problemas de sincronización comunes que hemos visto.

    Marketo los eventos de vista REST no se sincronizan con el error No se encontró el ID de tipo de actividad '000000'

    Esto sucede cuando se ha eliminado la actividad personalizada de Marketo asociada con la integración. Cuando se crea una integración de REST de Marketo, recuperamos la actividad, almacenamos su ID y usamos ese ID para sincronizar eventos. La solución (suponiendo que se haya creado una nueva actividad de Brightcove Video View) es desconectar y volver a crear la integración para recoger el ID de la nueva actividad.

    Ver eventos no sincronizados

    Cuando la integración está configurada correctamente con un MAP, la visualización de datos de un reproductor habilitado para Audience debe rellenar la página Toda actividad reciente y ver los eventos correspondientes a un cliente potencial conocido en el MAP respectivo se sincronizarán eventualmente con la herramienta.

    En primer lugar, asegúrese de que los datos de vista se muestren en la página Toda la actividad reciente :

    Si aparecen eventos de vista pero no están en el MAP, es posible que el usuario haya autorizado Audience con su MAP como usuario que no tiene permisos suficientes para importar datos de vista en el MAP a través de la API o que la integración no está configurada correctamente. La página principal de Audiencia muestra el estado de las operaciones de sincronización de eventos de vista recientes. Los comentarios deben estar disponibles allí para indicar por qué los eventos de visualización no se sincronizan con el MAP.

    Si aparecen eventos de vista en Audiencia, compruebe si la columna Estado está sincronizada. Esto indica que Audience pudo entregar correctamente los datos en su MAP. Si ve un estado de unknown_contact, esto significa que el usuario no era conocido y su MAP no puede recibirlo. La mayoría de los MAPs solo aceptan datos de terceros para usuarios conocidos. La solución es utilizar un formulario de prospecto personalizado creado en su MAP para asegurarse de que es realmente un usuario conocido.

    Si los eventos de visualización no aparecen en la lista, es posible que Audience no se haya configurado con un reproductor habilitado para Audiencia. Utilizando un navegador con herramientas de desarrollador habilitadas, visite una página que contenga el vídeo problemático y asegúrese de que ciertas llamadas HTTP son realizadas por el plugin Audience.

    Cuando el reproductor se carga, el plugin debe hacer una llamada al endpoint de configuración de Audience para obtener parámetros que determinen el comportamiento del complemento basado en la integración:

    Si esta llamada falta en el tráfico de red, significa que no se ha asignado una integración al reproductor. Para obtener información sobre cómo hacerlo, consulte Creación de un reproductor habilitado para audiencia-habilitado.

    Esta llamada se realiza usando JSONP. Si la respuesta JSONP es un objeto vacío, entonces es posible que el código de inserción sea incorrecto y especifique el ID de cuenta incorrecto. Se devolverá un objeto vacío desde este punto final si la solicitud de configuración no incluye parámetros de URL de la cuenta y del reproductor o si la cuenta especificada no se pudo recuperar de la base de datos específica del público (lo más probable es que la cuenta incorrecta se especificó en el código URL/incrustar y no se encontró en el base de datos).

    /**/ tipode jsonpCallback_78754666587 === 'función' && jsonpCallback_78754666587 ({});

    Si el objeto tiene un parámetro ID, pero le falta un parámetro de integración, el usuario no ha configurado una integración con su herramienta de automatización de marketing y no se registrarán los eventos de visualización.

    También tendrá problemas de sincronización de datos si se utilizó el código de incrustación estándar (iframe) al publicar el reproductor habilitado para audiencia-habilitado. Utilice el código de inserción avanzada (en página) cuando publique el reproductor. El código de publicación Standard Insert (iframe) no funcionará ya que el complemento Audience se ejecutará dentro de un iframe y, por lo tanto, no podrá acceder a las cookies de la página principal o a los parámetros de URL.

    Problemas relacionados

    El módulo Audience dice que no puede configurar una integración de Eloqua

    Compruebe los requisitos de Eloqua. Si utiliza la función de seguridad de la lista blanca IP, Audience no podrá comunicarse con Eloqua a menos que ponga en la lista blanca las direcciones IP especificadas para permitir el tráfico Audience - Eloqua. Póngase en contacto con el soporte técnico de Brightcove para obtener más información

    La audiencia dice que a mi actividad personalizada de REST de Marketo le faltan campos, pero cuando miro en Marketo, puedo ver que todos los campos existen

    Hay algunas cosas que comprobar:

    1. Los nombres de los campos deben coincidir exactamente. «Evento 1" no es lo mismo que «Event1".
    2. La actividad debe ser aprobada y no en estado de borrador.
    3. Si eliminó la actividad personalizada y la volvió a crear sin configurar una nueva integración de Marketo REST, debe desconectar la integración anterior y crear una nueva. Buscamos la actividad personalizada y almacenamos su valor de ID para cada integración. Si el ID cambia eliminándolo, no podremos recoger el nuevo automáticamente.
    4. Si se agregaron nuevos campos recientemente a la actividad, el cambio podría tardar hasta 24 horas en surtir efecto en el sistema de Marketo.
